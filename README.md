# クルマカルテ — 日本版・民間車両ヒストリーサービス(コンセプトデモ)

米国CARFAXに着想を得た、**民間運営の日本版車両履歴サービス**のプロトタイプです。
「クルマの物語(ストーリー)」を記録し、売るとき・買うときの信頼と価値に変えることを目指します。

**デモページ:** https://playmark0227-svg.github.io/carfax-japan/

## コンセプト

| 特徴 | 内容 |
|---|---|
| 認定店のみ初期設定 | ETCセットアップのように、車を購入した認定店など認定を受けた店舗でのみ車両登録が可能。なりすましのない確かなヒストリーの起点を作る |
| 自動ログアウトなし | 一度スマホでログインすればずっとログイン状態。半年ぶりに開いてもすぐ日記が書ける |
| 愛車日記 | 更新は半年に1回でOK。どう乗って、どう手入れしたかを気軽に記録。積み重なった日記が「大切にされてきた証明」になる |
| 認定工場の整備記録 | 詳細な整備記録は認定を受けた整備工場のみが記入(紙の記録簿の転記もOK)。第三者記録として買い手の信頼になる |
| AI分析 | 日記と整備記録をAIが解析し、「週末レジャー中心」「メンテ良好」など使われ方を推定。100点満点の減点方式スコアを減点理由つきで表示 |
| 売却レポート | ヒストリースコア・AI要約・物語のタイムラインをまとめた買い手向けレポートを発行 |
| 実車写真 | 車両カード・愛車アルバム・日記・レポートに実車写真を表示(日記には写真を添付可能) |
| ヒストリー共有 | Web共有 / X / LINE / リンクコピーで、読み取り専用の公開ヒストリーページを共有(URLに個人情報は含めない) |
| メンテナンス予定 | 記録から次回の車検・法定点検の期限をカウントダウン表示 |
| 信頼バッジ | メーター改ざん検知(走行距離の単調増加)・認定整備率などを自動判定して表示 |

## デモの使い方

- 初回アクセス時は、5年分の記録が入ったサンプル車両(マツダ ロードスター)を読み込んだ状態で **「マイカー」画面から** 始まります。そのまま日記・整備記録・AI分析・売却レポートを試せます
- 認定店での登録フローを見たい場合は、下部フッターの「デモデータをリセット」→ **「ホーム」**タブ →「初期設定をはじめる」から体験できます

### デモ用コード

| 用途 | コード |
|---|---|
| 認定店コード(初期設定) | `DEMO-1234` / `DEMO-5678` / `DEMO-9012` |
| 認定工場コード(整備記録の記入) | `FACT-1111` |

## 技術構成

- 静的サイト(HTML / CSS / Vanilla JS)— ビルド不要、GitHub Pagesでそのまま動作
- データはブラウザの localStorage に保存(サーバー送信なし)
- AI分析はデモ用のルールベース解析(製品版ではLLMによる文章解析を想定)
- 共有はバックエンド不要:公開用スナップショット(個人情報を除く)をURLの `#public?c=...` フラグメントに埋め込み、読み取り専用ページとして復元
- 車両写真はデモ用イメージ(提供: [Unsplash](https://unsplash.com))。`img/` に同梱
- 日記に添付した写真は canvas で縮小し dataURL として localStorage に保存

## ローカルでの起動

```bash
# 任意のHTTPサーバーで配信するだけ
python3 -m http.server 8000
# → http://localhost:8000
```
